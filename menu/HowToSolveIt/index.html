      어떻게 문제를 풀 것인가? #

      어떻게 문제를 풀 것인가? (How to solve it) 을 읽고 정리하기 위한 글 입니다.

      책이 너무 딱딱한 느낌이 들어 친구와 대화하듯 정리한 것입니다. (구어체)

      상당히 반복되는 내용이 많습니다. 중요한 것은 실제 문제에 적용해보며 체화하는 것입니다.

      수학에서 어떤 어려운 문제는 직선을 하나 그림으로써 상당히 쉽게 풀린다.
      그렇다면 이 직선을 그려보겠다는 생각은 어디서 나오는걸까?

      • 제목 그대로 문제를 어떻게 풀 것인가에 대한 고찰이다.
      • 정확히 말하면 어떻게 “수학적으로” 풀 것인가에 대한 것이다.
      • 프로그래밍이란 컴퓨터와 코드로 어떤 문제를 푸는 것인데 그것에 대한 이야기이다.

      1. 문제 이해하기 #

      • 구하려는 것이 뭐야?
      • 구하려는 것을 살펴봐
      • 조건은 만족될 수 있어?
      • 그림으로 그려봐
      • 조건을 여러 부분으로 나눠봐
      문제를 이해해야한다.

      2. 계획하기 #

      • 전에 본 적 있어?
      • 관련된 문제를 알고 있어?
      • 관련된 문제를 풀어봤으면 그것을 활용할 수 있을까?
      • 문제를 다른 말로 바꿔봐
      • 그래도 안된다면..
      • 자료에서 유용한 것을 찾아보자
      • 자료는 모두 썼어?
      구하려는 것과 자료 사이의 관계를 찾아보고 찾을 수 없다면 보조 문제를 고려해야 한다.
      이 단계에서 풀이에 대한 계획이 작성돼야 한다.

      3. 실행하기 #

      • 실행 전 생각을 검토해봐
      계획을 실행한다.

      4. 반성 #

      • 결과를 점검할 수 있어?
      • 다른 방법은 없을까?
      • 결과를 활용할 수 있나?
      • 귀납 / 수학적 귀납법
      풀이를 점검한다.

      문제를 푸는 과정은 다음과 같아!

      문제 이해하기 -> 계획하기 -> 실행하기 -> 반성

      이런 절차에 따라 문제를 푸는 것은 당연하지.

      아래는 각 단계를 진행할 때 도움이 될 설명이야.

      • 눈에 익히기 - 문제 이해하기

        • 문제의 전체적인 모습을 그려보며 친숙해지기
        • 잠깐 안봐도 머리에 남을 정도로
        • “답을 구하는 문제” = 구하려는 것, 자료, 조건
        • “증명 문제” = 가정, 결론

      • 도움이 되는 아이디어 찾기 - 계획하기

        • 문제의 핵심이 정리되었을때 시작
        • 이전에 얻은 지식과 연결점 찾기
        • 답을 구할수 있는 아이디어 찾기 (이 과정을 통해 문제에 대한 이해가 더 깊어짐)

      • 계획의 실행 - 실행하기

        • 주요한 관계를 확실히 이해 했고 해답의 아이디어가 있을 때 시작
        • 파악한 것을 확실히 하고 큰 단계와 작은 단계로 나눠 큰 단계 먼저 점검

      • 반성

        • 해답에 대해 다시 생각해보기
        • 단순화 해보기

      아이디어 사전 #

      • 거꾸로 생각하기
      • 교훈에 담긴 지혜
      • 결의, 희망, 성공
      • 귀납 / 수학적 귀납법
      • 귀류법 / 간접증명법
