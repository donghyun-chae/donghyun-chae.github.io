        SICP 1.14 #

        문제 #

        #lang sicp
(define (count-change n)
  (define (count amount kinds-of-coins)
    (cond ((= amount 0) 1)
          ((or (< amount 0) (= kinds-of-coins 0)) 0)
          (else (+ (count (- amount (value-of-coins kinds-of-coins)) kinds-of-coins)
                   (count amount (- kinds-of-coins 1))
                   )
                )
          ))
  
  (define (value-of-coins kinds-of-coins)
    (cond ((= kinds-of-coins 1) 50)
          ((= kinds-of-coins 2) 25)
          ((= kinds-of-coins 3) 10)
          ((= kinds-of-coins 4) 5)
          ((= kinds-of-coins 5) 1)))
  (count n 5)
  )

(count-change 11)

        1. 위 count-change 프로시저가 만들어내는 프로세스를 나무꼴로 그려라. #

        2. 프로세스가 거치는 단계 수와 기억 공간의 크기는 어떤 자람 차수를 보이는가? #

        문제풀이 #

        1. 위 count-change 프로시저가 만들어내는 프로세스를 나무꼴로 그려라. #


        6, 10의 가장 최하위 노드들의 가지 1개가 생략됨

        2. 프로세스가 거치는 단계 수와 기억 공간의 크기는 어떤 자람 차수를 보이는가? #

        우선 기억공간의 크기는 n=11 일때 15이고 n=12 일때 16이다. 즉 선형적으로 증가한다.
        빅오표기법으로는 O(n) 이다.

        프로세스가 자라나는 정도는 작은 것 부터 생각해나가는 것이 도움이 되는데, (count-change n 1) 의 경우에
        2n+1 의 크기이고 (count-change n 2) 은 (count-change n 1)이 n/5+1 개가 존재하기 때문에
        이차식이 만들어지고 (count-change n 2) 는 O(n^2) 의 복잡도를 갖는다.
        이것을 확장시키면 (count-change n 5)의 복잡도는 O(n^5) 이다.
