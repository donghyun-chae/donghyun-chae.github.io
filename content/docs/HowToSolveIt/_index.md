# 어떻게 문제를 풀 것인가?

수학에서 어떤 어려운 문제는 **직선을 하나 그림으로써** 상당히 쉽게 풀린다.  
그렇다면 이 직선을 그려보겠다는 **생각**은 어디서 나오는걸까?

{{< hint warning >}}

- 제목 그대로 문제를 어떻게 풀 것인가에 대한 고찰이다.
- 정확히 말하면 어떻게 "수학적으로" "소프트웨어적으로" 풀 것인가에 대한 것이다.
- 프로그래밍이란 컴퓨터와 코드로 어떤 문제를 푸는 것인데 그것에 대한 이야기이다.
  {{< /hint >}}

## **1. 문제 이해하기**

- 구해야하는 것, 자료, 조건 파악하기
- 조건이 만족 될 수 있는가? 충분한가? 나눠 볼 수 있는가?
- 그림과 기호 활용하기

## **2. 계획하기**

- 관련된 문제 떠올리기
- 문제를 다르게 바꿔보기
- 유사한 것을 떠올리고 필요하다면 변형하기

## **3. 실행하기**

- 매 단계를 점검하기

## **4. 반성**

- 결과를 점검하기
- 다른 방법 생각하기

---

<mark> 문제 이해하기 -> 계획하기 -> 실행하기 -> 반성 </mark>
이런 절차에 따라 문제를 푸는 것은 상식적이고 당연한 것이다.

아래는 조금 더 구체화 된 설명이다.

- **눈에 익히기 - 문제 이해하기**

  - 문제의 전체적인 모습을 그려보며 친숙해지기
  - 잠깐 안봐도 머리에 남을 정도로
  - "답을 구하는 문제" = 구하려는 것, 자료, 조건
  - "증명 문제" = 가정, 결론

- **도움이 되는 아이디어 찾기 - 계획하기**

  - 문제의 핵심이 정리되었을때 시작
  - 이전에 얻은 지식과 연결점 찾기
  - 답을 구할수 있는 아이디어 찾기 (이 과정을 통해 문제에 대한 이해가 더 깊어짐)

- **계획의 실행 - 실행하기**

  - 주요한 관계를 확실히 이해 했고 해답의 아이디어가 있을 때 시작
  - 파악한 것을 확실히 하고 큰 단계와 작은 단계로 나눠 큰 단계 먼저 점검

- **반성**
  - 해답에 대해 다시 생각해보기
  - 단순화 해보기
